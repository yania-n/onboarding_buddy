# **IT System Architecture — Tool Relationships**

The following diagram (maintained in Miro and exported quarterly to Confluence: IT Department > Architecture > Tool Ecosystem Map) shows how IT's core tools are interconnected:

| **Integration** | **Direction** | **What flows** |
| --- | --- | --- |
| **Workday → ServiceNow** | Automatic trigger | New hire and offboarding events trigger IT provisioning workflows |
| **ServiceNow → AWS / Azure AD** | Automated provisioning | Approved access requests automatically create accounts and roles |
| **GitHub → Terraform Cloud** | CI/CD trigger | Pull request merge triggers Terraform plan/apply in the relevant workspace |
| **Terraform Cloud → AWS / Azure / GCP** | Apply | Infrastructure changes deployed to cloud accounts |
| **AWS CloudTrail → Snowflake** | Nightly export | Security logs available for analytics and compliance queries |
| **Snowflake → Power BI** | DirectQuery + scheduled refresh | Analytics dashboards query Snowflake for live and daily-refresh data |
| **Datadog → PagerDuty** | Alert routing | Datadog alert conditions trigger PagerDuty incident creation and on-call notification |
| **Fivetran → Snowflake** | Continuous ingestion | SaaS and ERP source data landed in Snowflake raw layer every 15–60 minutes |
| **Wiz → Jira** | Automated ticket creation | Critical CSPM findings auto-create Jira tickets in the Cloud Security board |
